## 3. 新規ヘルパー関数（`functions.php`）

### 3.1 `lc_parse_price( $str )`
価格文字列を **円単位の整数** に正規化する。

| 入力 | 出力 |
|---|---|
| `"300万円"` | `3000000` |
| `"3万"` | `30000` |
| `"1,200"` | `1200` |
| 全角数字混在 | `mb_convert_kana` で半角化 |
| パース不能 | `0` |

### 3.2 `lc_get_fudo_terms( $taxonomy, $args = array() )`
物件タクソノミーのターム一覧を取得（`wp_cache` で 5 分キャッシュ）。
- グループ：`lc_fudo`
- キー：`lc_terms_<md5(taxonomy + args)>`

### 3.3 `lc_build_fudo_query_args( $filters )`
フィルタ配列から `WP_Query` 引数を構築する中核関数。

**対応フィルタ**：
- `kbn` — meta(`kbn` / `shubetsu`) と taxonomy(`fudo_kbn`) を OR で両対応
- `area[]` — `fudo_area` タクソノミー（存在時）または meta(`shozaichi` / `address` / `jusho`) を LIKE で
- `mado[]` — meta(`madori` / `layout`) を LIKE OR
- `price_min` / `price_max` — meta(`kakaku` / `price` / `bukken_kakaku`) を BETWEEN/`>=`/`<=`。万円単位の入力（1万未満）を自動で 10000 倍補正
- `tag` — `fudo_tag` タクソノミー（カンマ区切り対応）
- `orderby` — `date` / `price_asc` / `price_desc`
- `paged` / `posts_per_page` / `s`

### 3.4 `lc_get_fudo_filters_from_request()`
`$_GET` 全パラメータをサニタイズして `lc_build_fudo_query_args()` 用配列を返すラッパー。

### 3.5 `lc_render_property_block_card( $post_id, $tag_class, $tag_label, $display_items, $button_text )`
ブロック用に拡張した物件カード描画。`displayItems`（title/price/layout/area の表示有無）と `buttonText` を反映。

## 6. アーカイブ改修（`archive-fudo.php`）

従来のメインクエリの代わりに `lc_build_fudo_query_args()` + 独自 `WP_Query` を使用。

### 6.2 絞り込みチップ

|---|---|
| `kbn` | 種別 |
| `area[]` | エリア（配列パラメータも個別解除可） |

## 8. ブロック CSS の自動エンキュー（`functions.php`）

`blocks/css/*.css` を `glob()` で列挙し、`wp_enqueue_style()` で全件読み込み。

```php
add_action( 'wp_enqueue_scripts', function () {
    $files = glob( get_stylesheet_directory() . '/blocks/css/*.css' );
    foreach ( $files as $file ) {
        $name = basename( $file, '.css' );
        wp_enqueue_style( 'lc-block-' . $name, ..., array( 'lc-child-theme' ), LC_THEME_VERSION );
    }
}, 30 );
```

新規ブロック CSS を `blocks/css/` に置くだけで自動エンキューされる。

## 11. 運用方法

> **重要**：子テーマ独自のウィジェットエリア（`lc-sidebar` / `lc-top` / `lc-footer-1〜3`）は廃止し、
> 親テーマ／fudou プラグインの既存ウィジェットエリアを再登録する形に統合した（[Section 15](#15-ウィジェットエリア統合改訂版2026-05-17) 参照）。

### 11.1 トップページ（`top_widgets` ウィジェットエリア = fudou プラグイン）
管理画面 → **外観 → ウィジェット → トップページ (ホーム本文)** に以下を順番に配置：

1. **物件を探す**（`lc-property-search`）
2. **最新情報**（`lc-news-list`）
3. **賃貸物件**（`lc-rental-properties`）
4. **売買物件**（`lc-sale-properties`）

### 11.2 サイドバー（`sidebar-widget` ウィジェットエリア = 親テーマ）
管理画面 → **外観 → ウィジェット → サイドバー (物件検索)** に以下を配置：

1. **キーワードで物件を検索**（`lc-sidebar-search`）
2. **エリアから探す**（`lc-sidebar-areas`）
3. **設備・条件タグ**（`lc-sidebar-tags`）

### 11.3 フッター3カラム（`ftrnav-widget-{left,center,right}` = 親テーマ）
管理画面 → **外観 → ウィジェット → フッターナビゲーション：左/中央/右** に通常の WP メニュー／カスタム HTML ブロックを配置。

### 11.4 追加で利用可能な親テーマ既存エリア

| `front-before-widget` | Hero と本文の間 | 注目バナー、キャンペーン告知 |

## 14. 既知の留意点

- **`:has()` セレクタ**：theme.css で使用。Chrome 105+ / Safari 15.4+ / Firefox 121+ で対応。古いブラウザでは 3 カラム固定となるが致命的問題ではない。
- **キャッシュ**：`lc_get_fudo_terms()` の `wp_cache_set()` は永続キャッシュプラグイン（Redis/Memcached 等）がなければプロセス内キャッシュのみ。リクエスト跨ぎのキャッシュが必要なら `set_transient()` への切替を検討。
- **`fudo_kbn` taxonomy 不存在時**：種別は `meta(kbn / shubetsu)` の OR でフォールバック済み。
- **エディタプレビュー**：全ブロックで `ServerSideRender` を使用しているため、wp-admin 側で `wp.serverSideRender` がロードされる Gutenberg 環境が必要。

## 15. ウィジェットエリア統合（改訂版・2026-05-17）

### 15.1 改訂の経緯

初版では子テーマで以下の独自ウィジェットエリアを `register_sidebar()` していたが、いずれも親テーマ（SYN Ownd）／fudou プラグインの既存エリアと用途が重複していた：

| 旧（子テーマ独自） | 重複先 |
|---|---|
| `lc-sidebar` | `sidebar-widget`（親テーマ） |
| `lc-top` | `top_widgets`（fudou プラグイン） |
| `lc-footer-1` | `ftrnav-widget-left`（親テーマ） |
| `lc-footer-2` | `ftrnav-widget-center`（親テーマ） |
| `lc-footer-3` | `ftrnav-widget-right`（親テーマ） |

管理画面でウィジェットエリアが二重に表示され運用者が混乱するため、**案A：親テーマIDに揃える**方針で統合。

### 15.2 採用した戦略

子テーマ独自IDは登録せず、親テーマ／fudou プラグインで既に登録されているウィジェットエリアを `unregister_sidebar() → register_sidebar()` で **マークアップだけ上書き** する。
これにより以下を両立：

- **管理画面の重複解消**：同じ用途のエリアが1つに統一
- **既存 `.lc-widget` CSS の維持**：`before_widget` / `before_title` を子テーマ規約で再登録
- **親テーマの豊富なウィジェットエリア活用**：`*-before-widget` / `*-after-widget` などの拡張ポイントが使用可能に
- **fudou プラグイン互換**：fudou が想定する `top_widgets` を素直に使用

### 15.3 実装詳細（`functions.php`）

fudou プラグインは `widgets_init` 優先度10で `top_widgets` を登録するため、子テーマは**優先度11**で再登録：

```php
add_action( 'widgets_init', function () {
    $overrides = array(
        'sidebar-widget'       => array( 'name' => 'サイドバー (物件検索)', ... ),
        'top_widgets'          => array( 'name' => 'トップページ (ホーム本文)', ... ),
        'ftrnav-widget-left'   => array( 'name' => 'フッターナビゲーション：左 (サービス)', ... ),
        'ftrnav-widget-center' => array( 'name' => 'フッターナビゲーション：中央 (会社情報)', ... ),
        'ftrnav-widget-right'  => array( 'name' => 'フッターナビゲーション：右 (エリア)', ... ),
    );
    foreach ( $overrides as $id => $p ) {
        unregister_sidebar( $id );
        register_sidebar( array(
            'id'            => $id,
            'name'          => $p['name'],
            'description'   => $p['desc'],
            'before_widget' => '<div class="lc-widget %2$s" id="%1$s">',
            'after_widget'  => '</div>',
            'before_title'  => '<h3 class="lc-widget__ttl">',
            'after_title'   => '</h3>',
        ) );
    }
}, 11 );
```

### 15.5 親テーマ既存エリアの活用追加

`front-page.php` および `archive-fudo.php` で、親テーマの **コンテンツ前/後エリア** をテンプレートにフックとして追加：

| テンプレート | 追加されたフック |
|---|---|
| `front-page.php` | `front-before-widget`（Hero 直後）／`front-after-widget`（フッター直前） |
| `archive-fudo.php` | `archive-before-widget`（パンくず前）／`archive-after-widget`（フッター直前） |

これにより、管理者がコードを触らずに「アーカイブ上部に告知バナーを出す」「ホーム末尾に CTA を入れる」といった運用が可能。

## 17. classic-widgets 対応：ブロックを WP_Widget にラップ（2026-05-17）

### 17.1 発覚した問題

完全ウィジェット化後、管理者が「外観 → ウィジェット」で `top_widgets` にブロックを配置しようとしたが、

- ウィジェット管理画面が **従来UI（左ドラッグ式）** だった
- 「賃貸物件」「売買物件」などのブロックが **ウィジェット一覧に出てこない**
- フロントは黄色の「未設定」通知だけ表示

という状態だった。

### 17.2 原因

サイトでは [classic-widgets プラグイン](https://wordpress.org/plugins/classic-widgets/) が有効化されていた。
このプラグインは WordPress 5.8 以降の **ブロックベースウィジェットエディタを完全に無効化** するため、`register_block_type()` で登録した Gutenberg ブロックはウィジェットエリアに直接挿入できない。

### 17.3 対応：各ブロックを WP_Widget としてラップ

新規ファイル [`inc/class-lc-widgets.php`](inc/class-lc-widgets.php) を作成し、全7ブロックに対応する WP_Widget 派生クラスを実装：

| WP_Widget クラス | 表示名 | ラップ対象ブロック |
|---|---|---|
| `LC_Widget_Property_Search` | LC 物件検索フォーム | `lc-property-search` |
| `LC_Widget_Rental_Properties` | LC 賃貸物件 | `lc-rental-properties` |
| `LC_Widget_Sale_Properties` | LC 売買物件 | `lc-sale-properties` |
| `LC_Widget_News_List` | LC 最新情報 | `lc-news-list` |
| `LC_Widget_Sidebar_Search` | LC キーワード検索 | `lc-sidebar-search` |
| `LC_Widget_Sidebar_Tags` | LC タグクラウド | `lc-sidebar-tags` |
| `LC_Widget_Sidebar_Areas` | LC エリアから探す | `lc-sidebar-areas` |

### 17.4 設計上のポイント

#### 表示ロジックの完全共有
共通基底クラス `LC_Block_Widget extends WP_Widget` を作り、`widget()` メソッドで `$attributes = $instance` のまま対応する **`blocks/src/{slug}/render.php` を include**。
これによりブロック版とウィジェット版で **表示ロジックが完全に同一** となり、ブロック側の修正が自動的にウィジェット側にも反映される。

```php
public function widget( $args, $instance ) {
    $attributes = wp_parse_args( (array) $instance, $this->defaults() );
    echo $args['before_widget'];
    include $this->render_path();   // blocks/src/{slug}/render.php
    echo $args['after_widget'];
}
```

#### 属性の型推定によるサニタイズ
基底クラスの `update()` は `defaults()` の値の型（bool / int / array / string）から自動で適切なサニタイズを行う。
配列が複雑な場合（`LC_Widget_Property_Search` の `displayFields` 等）はサブクラスで `update()` をオーバーライド。

#### フォームUI
各ウィジェットの `form()` メソッドでブロック属性に対応した管理画面フォームを提供。タイトル・件数・並び順・チェックボックスなどブロック設定と同じ操作感。

### 17.5 ロード経路

`functions.php` 冒頭で `require_once` し、`widgets_init` の中で `register_widget()` を呼ぶ。

```php
// functions.php
require_once get_stylesheet_directory() . '/inc/class-lc-widgets.php';
```

```php
// inc/class-lc-widgets.php
add_action( 'widgets_init', function () {
    register_widget( 'LC_Widget_Property_Search' );
    register_widget( 'LC_Widget_Rental_Properties' );
    // ...
} );
```

### 17.6 UI 両対応

| 環境 | ブロック | WP_Widget |
|---|---|---|
| classic-widgets 有効（従来UI） | ❌ 挿入不可 | ✅ 「LC 〜」として表示・配置可能 |
| classic-widgets 無効（ブロックUI） | ✅ ブロック挿入可能 | ✅ 旧ウィジェットとしても配置可能 |

どちらの管理画面UIでも運用可能となり、プラグインの有効／無効に依存しない。

### 17.7 管理者向け通知の更新

`front-page.php` の管理者向け通知文を、案内するアイテム名を **「LC 物件検索フォーム」「LC 最新情報」「LC 賃貸物件」「LC 売買物件」** のウィジェット名に統一。

### 17.8 運用手順（最終形）

管理画面 → **外観 → ウィジェット → トップページ (ホーム本文)** に以下の WP_Widget を順番にドラッグ配置：
